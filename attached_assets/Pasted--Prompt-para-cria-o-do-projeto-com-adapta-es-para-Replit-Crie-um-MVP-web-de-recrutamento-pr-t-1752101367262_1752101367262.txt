🚀 Prompt para criação do projeto (com adaptações para Replit)
Crie um MVP web de recrutamento, prático, simples e funcional, com foco em excelente experiência de usuário, conforme os requisitos abaixo. O projeto será hospedado e rodado no Replit, portanto, atente-se às especificidades do ambiente.

🔐 Autenticação e perfis
Empresa

Cadastro/Login com: nome da empresa, email, senha (hash), e CNPJ (validado e único no banco).

Home da empresa: dashboard com lista de vagas, botão “Cadastrar nova vaga” e ranking de candidatos.

Recuperação de senha via email com token expirável.

Acesso ao currículo completo e contatos (telefone, email) dos candidatos.

Candidato

Cadastro/Login com: nome completo, email, senha (hash), LinkedIn e telefone obrigatório.

Home do candidato: painel exibindo pretensão salarial, botão "Enviar currículo", e histórico de candidaturas e posições no ranking.

Recuperação de senha via email.

Onboarding guiado, destacando o que fazer em cada etapa.

📝 Cadastro de vagas
Formulário para empresas criarem vagas com:

Título (obrigatório)

Descrição (obrigatório)

Requisitos (palavras-chave separadas por vírgula)

Salário oferecido (numérico)

Validação frontend/backend com mensagens claras.

Indicação de progresso no preenchimento.

Interface responsiva (Tailwind ou Bootstrap minimal).

📄 Upload e processamento do currículo
Suporte a PDF (PyPDF2) e DOCX (python-docx), até 5 MB, com validação de tipo e extensão.

Barra de progresso durante upload.

Extrai automaticamente:

Experiência profissional

Competências / Skills

Sobre / Resumo profissional

Exibe para o usuário:

✔ Campos extraídos com sucesso

✖ Campos não encontrados

Permite:

Completar manualmente dados faltantes.

Reenviar outro currículo para nova tentativa.

🤖 Matching e ranking
Compara pretensão salarial (candidato) e salário oferecido (vaga) para ajustar score.

Conta quantos requisitos da vaga aparecem no texto extraído.

Usa Hugging Face (sentence-transformers) para similaridade textual (cosine similarity em %).

Cache local dos embeddings (SQLite ou arquivos) para otimizar consultas.

Configuração via .env de quantas vagas mostrar (ex: TOP_JOBS=3).

🔁 Fluxo do candidato
Preenche dados pessoais, LinkedIn, telefone e pretensão salarial.

Faz upload do currículo.

Visualiza dados extraídos e pode complementar manualmente ou reenviar outro arquivo.

Recebe as top N vagas mais compatíveis.

Escolhe vaga para se candidatar.

Sistema registra candidatura, recalcula ranking e mostra score e posição.

Pode desfazer candidatura antes do cutoff configurado.

🏢 Fluxo da empresa
Visualiza vagas cadastradas e botão para criar novas.

Vê lista rankeada de candidatos para cada vaga.

Recebe notificações (email ou badge) para novas candidaturas ou mudança no Top 3.

Acessa currículo completo e contatos dos candidatos.

💽 Banco de dados e infraestrutura no Replit
Usar SQLite para persistência (arquivo db.sqlite3 no projeto).

Tabelas:

empresas (id, cnpj UNIQUE, nome, email, senha_hash)

candidatos (id, nome, email, senha_hash, linkedin, telefone, pretensao_salarial)

vagas (id, empresa_id, titulo, descricao, requisitos, salario_oferecido)

candidaturas (id, candidato_id, vaga_id, score_req, score_nlp, posicao)

Use o painel de Secrets do Replit para definir variáveis ambiente (em vez do .env):

MODO_IA=local|huggingface

TOP_JOBS=3

Configurações SMTP para email.
